独酌雅居

“独酌雅居”是我为自己这个项目命的名字，是用electron + vue + lowdb做的一款具有离线功能的，PC段的桌面应用。

项目由来

为什么要做这样一个东西出来呢？其实是我在日常的项目开发中自己所遇到的一个痛点，我不知道别的程序员有没有，可能是我的层次太低了，所以我会产生这样的需求。

在我的工作中经常会发生这样的事情，有些代码片段是不需要我们去记忆的，用到的时候记得拿过来用就行了，但是经常性的，我们会很辛苦的去别的项目里找代码，拷贝代码，这样的过程给我带来了很大的不方便，所以我就想做一个这样的应用来存储工作中可能经常要用到，但是不需要记忆，有的时候找起来又比较麻烦的代码，将这些代码统一的放到一个应用里管理，岂不是很方便。

第二个我在写程序的时候所遇到的痛点就是项目文件夹的管理了，有的时候项目一多，找起来就比较麻烦，虽然用快捷键放到桌面上还是比较方便的，但是我想放到一个专门的应用里管理，清晰明了的展现出一个个项目，一个个分类，应该是一件很方便的事情。

基于以上我在实际的工作中所遇到的痛点，所以我想到了自己做出一款这样的应用来，顺便也挑战一下自己，锻炼一下自己的能力。

为什么选择Electron + vue + lowdb

其实类似这样可以将前端页面，打包成桌面应用的工具还有一个选择，那就是NW，不过我选择了Electron，其实也没别的什么原因，只是我先接触的是electron，正好vscode也是用electron打造的，我挺喜欢vscode这款编辑器的，想来electron也不会太差。

什么是Electron

Electron 可以让你使用纯 JavaScript 调用丰富的原生 APIs 来创造桌面应用。你可以把它看作是专注于桌面应用而不是 web 服务器的，io.js 的一个变体。

这不意味着 Electron 是绑定了 GUI 库的 JavaScript。相反，Electron 使用 web 页面作为它的 GUI，所以你能把它看作成一个被 JavaScript 控制的，精简版的 Chromium 浏览器。

选择vue的理由其实很简单，因为vue本身不难，官网的api和一些入门的知识都挺全的，react给我的感觉学习曲线会更陡，没办法，我就是这么怂，肯定挑更加简单一点的。

lowdb是可以在本地运行的，非常轻量级的数据管理工具，相当于一个数据库了，只是很轻，非常非常轻，之前我一直在线上的数据库和离线的本地数据库之间犹豫不决，最终我还是选择了本地的数据处理方法，就让应用离线运行吧，毕竟我这个穷屌丝还没钱买服务器。
